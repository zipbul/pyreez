# 숙의 시스템 개선 계획 (v2)

## 배경

5개 모델(DeepSeek-R1, Grok-4, o3, Gemini-2.5-Pro, Claude-Sonnet-4.6)이 3라운드 debate를 통해
"truncated normal이 clamp보다 기대값 inflation을 줄인다"고 만장일치 합의했으나, 실제 구현 후 테스트에서
**수학적으로 반대**임이 밝혀졌다 (clamp E~195, truncated E~312).

5개 모델이 5라운드 추가 debate로 이 문제의 해결책을 논의했으나,
전부 동일한 프레임("검증 레이어 추가") 안에서만 사고했다.

### 근본 원인

1. **자세의 부재** -- 확신 없이 단언, 전제를 의심하지 않음, 대안을 탐색하지 않음, 스코프가 좁아짐
2. **능력의 한계** -- 모든 모델이 동일 학습 데이터에서 동일 오개념을 공유. 자세가 완벽해도 공유 맹점은 돌파 불가

따라서 **자세(프롬프트)와 구조(엔진) 두 계층 모두** 필요하다.

### v1에서의 교훈

v1(16개 항목)을 3개 서브에이전트 + GPT-5/GPT-5.2/Claude Opus 4.6 숙의 + 웹 조사로 엄격 검토한 결과:

- 16개 항목 중 상당수가 특정 실패(수학 오답 만장일치)에 과적합
- 3-모드 분류(검증/논증/발산)가 범용적이지 않음 (윤리, 해석, 예측, 규범 과제 누락)
- 메타인지(#5)는 구조적으로 불건전 (LLM의 Dunning-Kruger)
- 의무 시뮬레이션(#11)은 같은 오개념의 코드가 거짓 권위를 부여
- 심박 DA(#10), 적층 규칙(#15)은 ROI 부족 또는 역효과
- 가장 효과적인 접근은 무거운 인프라가 아닌 **토론의 자세와 역할 구조**

10개 분야(CIA ACH, Strong Inference, 소크라테스 방법, Six Thinking Hats, 지적 겸손 연구,
Steel-manning, 적대적 협업, Delphi, 티벳 불교 논쟁, 배심 숙의 연구)의 교차 검증으로
범용 원칙을 도출하여 v2를 설계했다.

---

## 설계 철학: 4계층 구조

```
계층 1: 보편 자세 원칙 (모든 참여자 공통 프롬프트)
         비용 0, 즉시 적용. 대부분의 자세 문제를 해결.

계층 2: 스탠스 분화 (워커 역할 분화)
         제안자 / 비판자 / 탐색자. 구조적 사고 다양성 확보.

계층 3: 리더 역할 재정의
         합의자 -> 검증자. 워커 주장의 독립 검증 + 누락 탐색.

계층 4: 구조적 안전망 (엔진 레벨)
         자세로 해결 불가능한 공유 맹점을 위한 최후 방어선.
```

---

## 계층 1: 보편 자세 원칙

모든 워커와 리더에게 공통 적용되는 8개 원칙.
10개 분야에서 교차 검증된 범용 원칙으로, 특정 도메인에 한정되지 않는다.

### 원칙 1: 증거-확신 정직성

> 모든 진술을 {관찰된 사실 / 추론 / 가설}로 구분하라.
> 확신도를 명시하라. 근거 없이 단언하지 마라.

- **출처**: Toulmin 모델, 지적 겸손 연구, 논증 품질 프레임워크(DQI)
- **해결하는 문제**: 거짓을 사실처럼 말하는 것
- **메커니즘**: 주장-근거-보증-반박조건을 명시하면 근거 없는 단언이 구조적으로 불가능
- **실패 모드**: 형식적 충족("확신도 90%"라고 쓰지만 실제 보정 안 됨). 그러나 형식적 충족만으로도 검증자(리더)가 근거를 추궁할 진입점이 생김

### 원칙 2: 경쟁 대안 유지

> 3개 이상의 경쟁 가설/접근법을 먼저 생성하라.
> 하나에 애착하지 마라. 확인하는 증거가 아닌 식별하는 증거를 찾아라.

- **출처**: Strong Inference(Platt 1964), CIA ACH, Chamberlin의 다중 작업 가설 방법
- **해결하는 문제**: 스코프 좁아지는 것, 터널 비전
- **메커니즘**: 단일 가설에 대한 "지적 애착"(Chamberlin)이 확증 편향의 근본 원인. 여러 가설을 동시에 유지하면 증거 해석이 균형적으로 변함
- **실패 모드**: 피상적 대안 생성(사실상 같은 접근의 변형 3개). 리더가 대안의 실질적 차이를 검증해야 함

### 원칙 3: 사고 모드 분리

> 생성 모드와 판단 모드를 섞지 마라.
> 생성할 때는 비판을 멈추고, 판단할 때는 철저히 하라. 현재 어떤 모드인지 선언하라.

- **출처**: Six Thinking Hats(De Bono), 디자인 싱킹(IDEO)
- **해결하는 문제**: 창의성 사망
- **메커니즘**: 비판이 생성을 실시간으로 죽이는 것은 실험적으로 확인된 사실. 모드를 분리하면 각 모드가 최대 효율로 작동
- **실패 모드**: 모드 선언이 형식적이 됨. 엔진 레벨에서 라운드별 모드를 강제하면 해결

### 원칙 4: 독립적 초기 판단

> 다른 참여자의 결론을 보기 전에 자기 견해를 먼저 형성하라.
> 업데이트는 새로운 증거/논거에 의해서만 하라. 단순 동조 금지.

- **출처**: Delphi 방법, Wisdom of Crowds(Surowiecki)
- **해결하는 문제**: 그룹싱크, 앵커링
- **메커니즘**: 독립성이 깨지면 오류가 상관되어 집단지성이 무너짐. 현재 엔진의 diverge 페이즈가 이미 이를 구현(워커 병렬 응답)
- **실패 모드**: 모델들이 동일 학습 데이터를 공유하므로 "독립적" 판단이 이미 상관됨. 이것은 자세로 해결 불가 -> 계층 4(구조적 안전망)로 보완

### 원칙 5: Steel-man 후 반박

> 반박 전에 상대 주장을 가장 강한 형태로 재진술하라.
> 허수아비가 아닌 강철인간을 이겨야 진짜 반박이다.

- **출처**: Rapoport 규칙, Dennett, John Stuart Mill
- **해결하는 문제**: 피상적 비판, 입장 고착
- **메커니즘**: 상대 입장을 가장 강하게 만든 후에도 성공하는 반박만이 실질적 정보를 생산. "자기 입장만 아는 사람은 그 입장조차 모른다"(Mill)
- **실패 모드**: Steel-man 형식을 충족하면서 실질적으로 왜곡. 리더가 재진술의 충실도를 검증해야 함

### 원칙 6: 증거 우선, 입장은 나중에

> 증거와 논리를 먼저 검토하라. 결론을 먼저 내리지 마라.
> 토론을 "경쟁하는 결론" 중심이 아닌 "문제와 증거" 중심으로 구조화하라.

- **출처**: 배심 숙의 연구(Hastie, Pennington), 구성적 논쟁(Johnson & Johnson)
- **해결하는 문제**: 진영화, 확증 편향
- **메커니즘**: 조기 투표 -> 진영 형성 -> 선택적 증거 활용이라는 인과 경로가 실증됨
- **실패 모드**: "증거"가 LLM의 학습 데이터에 기반하므로 증거 자체가 틀릴 수 있음 -> 계층 4(외부 도구)로 보완

### 원칙 7: 마음 바꿀 조건 사전 선언

> 자기 입장이 틀릴 수 있는 조건을 미리 밝혀라.
> 입장을 바꿀 때는 왜 바꿨는지 명시하라. 골대를 옮기지 마라.

- **출처**: 적대적 협업(Kahneman), 초예측(Tetlock), 과학적 사전등록
- **해결하는 문제**: 사후 합리화, 끝없는 논쟁
- **메커니즘**: 사전 선언이 없으면 어떤 증거에도 기존 입장 유지 가능. 사전 등록은 골대 이동을 구조적으로 차단
- **실패 모드**: 모호한 사전 선언("강력한 반증이 나오면"). 구체적이고 검증 가능한 조건을 요구해야 함

### 원칙 8: 자아와 신념의 분리

> 틀리는 것은 실패가 아니라 진보다.
> 논리를 공격하되 사람을 공격하지 마라. 교정을 환영하라.

- **출처**: 티벳 불교 논쟁, 지적 겸손 연구, 소크라테스적 무지
- **해결하는 문제**: 방어적 자세, 감정적 고착
- **메커니즘**: 자아가 신념에 묶이면 반증을 위협으로 인식. 분리하면 증거에 대한 개방성 증가
- **LLM에서의 의미**: LLM에게 "자아"는 없지만, RLHF로 인한 "자기 답변 방어 경향"은 있음. 이 원칙은 그 경향을 상쇄

## 계층 2: 스탠스 분화

8개 원칙을 모든 워커에게 동일 적용하되, 각 워커에게 다른 강조점을 부여한다.
스탠스는 **고정이 아닌 강조**. 비판자도 대안을 제시할 수 있고, 제안자도 자기 제안을 비판할 수 있다.

### 제안자 (Proposer)

- **강조 원칙**: #2(경쟁 대안), #3(생성 모드)
- **핵심 지시**: 여러 접근법을 제시하라. 자기 검열하지 마라. 실현 가능성은 나중에 판단한다. 기존 틀 밖의 접근도 포함하라.
- **해결하는 문제**: 스코프 좁아지는 것, 창의성 사망
- **주의점**: 원칙 #1(확신도 명시)은 여전히 적용. "검증되지 않은 아이디어"와 "확립된 방법"을 구분

### 비판자 (Critic)

- **강조 원칙**: #5(Steel-man 후 반박), #6(증거 우선)
- **핵심 지시**: 각 주장의 근거 구조를 검토하라. 반박 전에 가장 강한 형태로 재진술하라. 결론이 아닌 논거의 약점을 공격하라. 숨겨진 가정을 찾아라.
- **해결하는 문제**: 거짓을 사실처럼 말하는 것, 피상적 합의
- **주의점**: 임무는 "틀렸다"가 아니라 "이 근거가 충분한가?". 결론을 반대하는 게 아니라 논증 구조를 시험하는 것

### 탐색자 (Scout)

- **강조 원칙**: #2(경쟁 대안), #7(마음 바꿀 조건)
- **핵심 지시**: 우리가 놓치고 있는 관점, 고려하지 않은 이해관계자, 탐색하지 않은 영역을 찾아라. 현재 논의의 프레임 자체를 의심하라. 다른 분야에서 유사한 문제를 어떻게 풀었는지 탐색하라.
- **해결하는 문제**: 스코프 좁아지는 것, 프레임 고착
- **주의점**: 가치는 관련성 있는 외부 시각을 가져오는 것. 무관한 것을 끌어오는 것이 아님

### 스탠스 배분 규칙

- 2인 팀: 제안자 + 비판자
- 3인 팀: 제안자 + 비판자 + 탐색자
- 4인+ 팀: 제안자 복수 가능, 비판자/탐색자 각 최소 1인

## 계층 3: 리더 역할 재정의

**현재**: "워커 응답을 비교하고 최선을 종합하라" (합의자)
**변경**: 검증자 + 종합자 + 스코프 감시자

### 의무 1: 독립 검증

워커의 사실 주장을 액면 그대로 받지 마라. 근거가 충분한가, 논리에 비약이 없는가,
검증되지 않은 가정은 없는가를 독립적으로 판단하라. 다수 동의는 정당성의 근거가 아니다.

- 워커 전원이 동의해도 근거가 부실하면 거부할 수 있어야 함
- "왜 이렇게 생각하는가?"를 추궁하는 소크라테스적 역할

### 의무 2: 누락 탐색

워커들이 탐색하지 않은 영역이 있는가? 논의의 프레임이 적절한가?
더 넓은 맥락에서 볼 때 빠진 관점이 있는가?

- 워커 응답의 범위에 갇히지 않음
- 원칙 #2(경쟁 대안)를 메타 레벨에서 적용

### 의무 3: 정직한 종합

진짜 합의와 거짓 합의를 구분하라. 해결된 것과 미해결인 것을 명확히 분리하라.
불확실성을 숨기지 마라.

- "합의에 도달했다"가 아니라 "이 부분은 합의, 이 부분은 미해결, 이 부분은 추가 검증 필요"
- 워커들의 확신도가 높을수록 더 엄격하게 근거를 검증
- 만장일치는 안심이 아니라 주의 신호

## 계층 4: 구조적 안전망

자세(계층 1-3)로 해결 불가능한 문제를 위한 엔진 레벨 메커니즘.
"같은 자로 자기를 재는" 한계를 돌파하기 위한 최후 방어선.

### 안전망 1: 검증 모드

- **조건**: 계산 가능한 문제가 식별되었을 때
- **동작**: 토론 대신 코드 실행으로 검증
- **설계 원칙**:
  - 검증 가능성은 이진이 아닌 스펙트럼
  - 저비용/고신뢰 검증(산술, 결정적 코드)과 고비용/가정의존 검증(Monte Carlo)을 구분
  - 후자의 결과는 ground truth가 아닌 증거로 취급
- **핵심 제약**: 주장한 모델과 검증 코드를 작성하는 모델을 분리하거나, 최소한 검증 코드에 주장의 결론을 노출하지 않아야 함 (블라인드 검증)

### 안전망 2: 조건부 DA 트리거

- **트리거 조건** (3가지 복합, 단순 만장일치가 아님):
  1. 만장일치 또는 강한 합의
  2. 추론 경로의 유사성 (같은 결론 + 같은 논거 = 의심, 같은 결론 + 다른 논거 = 건강)
  3. 해당 영역의 LLM 약점 이력 (BT 점수에서 해당 차원의 sigma가 높은 경우)
- **동작**: 추가 라운드에서 DA 역할 모델이 합의에 대한 체계적 반론 제시
- **핵심 제약**: DA의 역할은 "반대 결론 주장"이 아닌 "논증 구조 공격"

### 안전망 3: 외부 도구 연결

- **목적**: LLM끼리 검증하는 한계 돌파
- **범위**: 코드 실행, 웹 검색, 계산기, DB 조회 등
- **동작**: 리더 또는 워커가 "이 주장은 외부 검증이 필요하다"고 판단 시 도구 호출
- **핵심 제약**: 도구 결과도 맥락 없이 맹신하면 안 됨. "이 시뮬레이션 결과는 X 가정 하에서 유효하다"

### 안전망 4: 숙의 학습

- **데이터 소스**: 외부 피드백(사용자 평가, 후속 검증 결과), 숙의 간 불일치 패턴
- **학습 대상**: 영역별 LLM 신뢰도, 반복되는 실패 패턴, DA 트리거 보정
- **핵심 제약**: 자기 평가가 아닌 외부 ground truth에만 기반
- **부트스트래핑**: 초기에는 인간 큐레이션 불신 규칙으로 시작, 데이터 축적 후 점진적 자동화

## 구현 우선순위

### Phase 1: 프롬프트 레벨 (즉시)

- 8개 원칙을 워커/리더 시스템 프롬프트에 반영
- 스탠스 분화 (제안자/비판자/탐색자) 프롬프트 설계
- 리더 프롬프트 재정의 (합의자 -> 검증자)

### Phase 2: 엔진 레벨 (단기)

- 스탠스 할당 로직 (팀 크기별 자동 배분)
- 리더 독립 사고 지원 (리더가 워커와 다른 프롬프트로 diverge 참여)
- 추론 경로 유사성 감지 (DA 트리거 조건 2)

### Phase 3: 인프라 레벨 (중기)

- 검증 모드 (블라인드 코드 실행 샌드박스)
- 외부 도구 연결 (검색, 계산기, DB)
- 조건부 DA 전체 구현

### Phase 4: 학습 레벨 (장기)

- 숙의 학습 파이프라인 (외부 피드백 -> 패턴 축적 -> DA 트리거 보정)
- 영역별 LLM 신뢰도 맵 자동 갱신
