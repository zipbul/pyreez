# Multi-Model Deliberation 리서치

> 최종 업데이트: 2026-04-05
> 핵심 구분: pyreez는 이종 모델(Claude, GPT, Gemini, Grok 등)을 동시에 사용하는 **멀티모델** 시스템. 같은 모델 복제본을 돌리는 멀티에이전트와 다름.
> 검증 기준: 2개 이상 독립 소스 확인 = "교차 검증됨". 나머지 = "unverified".

---

## 교차 검증된 핵심 발견

### 1. 이종 모델이 동종 모델을 일관되게 이긴다 (3개 소스)

- **Mixture-of-Agents (MoA)** (Together.ai, ICLR 2025): "responses generated by heterogeneous models contribute significantly more than those produced by the same model". 오픈소스 LLM만으로 AlpacaEval 2.0에서 65.1% (GPT-4 Omni 57.5% 초과). [arXiv:2406.04692](https://arxiv.org/abs/2406.04692)
- **X-MAS** (arXiv:2505.16997, 2025): 27개 LLM, 5개 도메인, 170만+ 평가. 동종→이종 전환 시 MATH +8.4%, AIME **+47%**. "구조 재설계 없이 이종 LLM으로 전환만으로 성능 대폭 향상". [arXiv](https://arxiv.org/abs/2505.16997)
- **ReConcile** (ACL 2024): Claude + GPT-4 + Bard 라운드테이블. "the diversity originating from different models is critical to its superior performance". 단일 모델 대비 최대 11.4% 향상. [arXiv:2309.13007](https://arxiv.org/abs/2309.13007)

**시사점**: pyreez의 이종 모델 접근이 연구적으로 강하게 지지됨. 같은 모델 복제보다 다른 모델 조합이 핵심 가치.

### 2. MAD는 최강 참여자를 초과할 수 없다 (2개 소스)

- **"Can LLM Agents Really Debate?"** (arXiv:2511.07784, 2025): 통제 실험. MAD는 가장 강한 참여자 정확도를 초과 불가. **약하거나 과신하는 에이전트가 팀 결과를 저하**. 고유 추론 능력 + 그룹 다양성이 성공의 지배적 요인.
- **Google Scaling Laws** (arXiv:2512.08296, 2026): Capability Saturation — 단일 에이전트 기준선 >45%에서 멀티에이전트 수확 체감/음수.

**시사점**: 약한 모델을 포함하면 전체 품질이 떨어진다. 팀 구성 시 모델 품질 하한선이 중요. 강한 모델끼리 조합해야 시너지.

### 3. 순응(sycophancy/conformity)이 멀티모델 토론의 최대 적 (3개 소스)

- **Free-MAD** (arXiv:2509.11035, 2025): LLM의 inherent conformity로 초기에 옳은 에이전트가 다수의 오답에 동조. Anti-conformity 메커니즘 + 전체 궤적 평가로 해결.
- **CONSENSAGENT** (ACL 2025 Findings): LLM의 아첨(sycophancy)이 비판적 참여를 억제. 동적 프롬프트 정제로 완화. 6개 벤치마크에서 베이스라인 초과. [ACL](https://aclanthology.org/2025.findings-acl.1141/)
- **"Can LLM Agents Really Debate?"**: 다수 압력이 독립적 수정을 억제.

**시사점**: 현재 pyreez의 프롬프트 수준 ANTI_CONFORMITY는 부족할 수 있다. 구조적 anti-conformity(격리, 궤적 평가, 선택적 공유)가 필요.

### 4. 토론 라운드를 늘리는 게 항상 낫지 않다 (3개 소스)

- **"Voting or Consensus?"** (ACL 2025): 투표 전 토론 라운드 증가 → 성능 저하.
- **Free-MAD**: 단일 라운드 토론만으로 다중 라운드 MAD 초과 성능.
- **Google Scaling** (2026): 순차적 추론에서 모든 멀티에이전트 변형 39-70% 저하.

**시사점**: 라운드 수를 늘리면 conformity가 강해지고 비용만 증가할 수 있다. 최소 라운드 + 강한 집계가 더 나을 수 있음.

### 5. 집계 방식이 토론 구조보다 중요하다 (2개 소스)

- **"Debate or Vote"** (NeurIPS 2025 Spotlight): majority voting만으로 MAD 성능 향상의 대부분 설명. debate은 martingale — 기대 정확도 불변.
- **"Voting or Consensus?"** (ACL 2025): 추론 → voting (+13.2%), 지식/분석 → consensus (+2.8%).

**시사점**: 토론 구조를 정교화하는 것보다 **어떻게 집계하느냐**가 더 큰 레버리지. 태스크별 집계 분화 필수.

### 6. Sparse 공유 > Full 공유 (3개 소스)

- **Sparse MAD** (EMNLP 2024): sparse ≥ full 정확도, 토큰 40%+ 절감.
- **EIB-Learner** (EMNLP 2025): 중간 sparse가 error propagation 억제 + information diffusion 보존에 최적.
- **Google Scaling** (2026): 독립 에이전트 17.2x 오류 증폭 vs 중앙 조정 4.4x.

**시사점**: 모든 워커가 모든 출력을 보는 현재 구조(full mesh)를 선택적 공유로 전환해야 한다.

### 7. 태스크별 최적 구조가 다르다 (3개 소스)

- **MacNet** (ICLR 2025): chain=코드, tree=창작.
- **G-Designer** (ICML 2025): 태스크 적응형 토폴로지 자동 설계. 고정 대비 우월.
- **FusionFactory** (arXiv:2507.10540, 2025): thought-level fusion > query-level > model-level. 14개 벤치마크에서 최고 개별 LLM 초과.

**시사점**: 호스트가 태스크에 따라 기법을 선택하는 구조가 지지됨.

## unverified (단일 소스)

### 8. ICE — 반복적 합의 앙상블

- **Iterative Consensus Ensemble** (2025, Computers in Biology and Medicine): 여러 LLM이 반복적으로 서로의 출력을 검토하며 합의. PhD 레벨 추론(GPQA-diamond) 46.9% → 68.2% (상대 45%+ 향상). 특수 보상 모델 불필요.
- [ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S0010482525010820)

### 9. A-HMAD — 적응형 이종 모델 Debate

- **Adaptive Heterogeneous MAD** (Springer, 2026): 역할 특화(Verifier, Solver) + 학습된 합의 모듈. 4-6% 향상, 사실 오류 30%+ 감소.
- [Springer](https://link.springer.com/article/10.1007/s44443-025-00353-3)

### 10. DMAD — 추론 방법 다양화

- **Diverse MAD** (ICLR 2025): 에이전트별 다른 추론 접근법 할당. "fixed mental set" 파괴. MAD보다 적은 라운드.
- [ICLR 2025](https://proceedings.iclr.cc/paper_files/paper/2025/hash/3de667dab3b3d812583abc0a786139a0-Abstract-Conference.html)

### 11. DeePEn — 이종 모델 확률 분포 퓨전

- **DeePEn** (NeurIPS 2024): 서로 다른 토크나이저의 확률 분포를 relative representation으로 매핑 후 집계. training-free. 약한 모델도 강한 모델에 기여 가능.
- [arXiv:2404.12715](https://arxiv.org/abs/2404.12715)

### 12. RA-CR — Rank-Adaptive 프로토콜

- **arXiv:2603.28813 (2026)**: 외부 judge가 라운드마다 워커 랭킹, 순서 재배치, 최하위 침묵. CR 대비 빠른 수렴.

### 13. BCCS — Belief-Calibrated Consensus

- **OpenReview 2026**: 최적 협업자 선택 + belief 보정. MATH +2.23%, MMLU +3.95%.

### 14. DER — Dynamic Ensemble Reasoning

- **IJCAI 2025**: LLM 앙상블을 MDP로 모델링, RL로 최적 라우팅. Knowledge Transfer Prompt로 모델 간 보완적 지식 전달. 파라미터 7배 감소.
- [arXiv:2412.07448](https://arxiv.org/abs/2412.07448)

### 15. Agreement Intensity 조절

- **ICML 2024**: 90% agreement 설정이 의료 QA에서 최대 15% 향상. 다른 도메인 미검증.

## pyreez 기법에 대한 멀티모델 관점 최종 판단

| 기법 | 판단 | 근거 |
|------|------|------|
| **공유 수렴** | 약화 | voting만으로 대부분 설명됨 (NeurIPS 2025). 라운드 증가가 오히려 해로움 (ACL 2025). sparse 전환 필수 |
| **대립 토론** | 조건부 유지 | 적당한 대립만 유효. 이종 모델 자체가 자연 다양성 제공하므로 강제 대립의 추가 가치 불확실 |
| **호스트 심문** | 유지 | Star 토폴로지. error amplification 최소 (Google: 4.4x). 프로덕션 검증 |
| **순차 정제** | 제한적 유지 | 순차 추론 39-70% 저하 (Google 2026). 병렬 분할 가능 산출물에만 |
| **평가 채점** | **강화** | voting이 debate보다 효과적. confidence-weighted voting + 태스크별 집계 분화 |
| **레드팀** | 유지 (낮은 우선순위) | 직접 연구 부족하나 비대칭 역할 구조는 A-HMAD에서 지지 |

### 연구가 시사하는 설계 원칙

1. **이종 모델 자체가 최대 무기**: 구조 정교화보다 모델 다양성 확보가 더 큰 레버리지 (MoA, X-MAS)
2. **약한 모델 배제**: 팀에 약한 모델 포함 시 전체 품질 저하. 품질 하한선 필수 (Can LLMs Debate)
3. **최소 라운드**: 라운드 증가 = conformity 증가 + 비용 증가. 1-2라운드 + 강한 집계가 최적 (Free-MAD, ACL 2025)
4. **구조적 anti-conformity**: 프롬프트만으로 부족. 격리/선택적 공유/궤적 평가 필요 (Free-MAD, CONSENSAGENT)
5. **집계 > 토론**: 토론 구조보다 집계 방식이 더 큰 영향 (NeurIPS 2025)
6. **태스크별 기법 선택**: 고정 기법이 아니라 태스크에 따라 자동 선택 (G-Designer, MacNet)
